# 음성 뉴스 스크랩 — 접근성 프로토타입

시력이 서서히 저하되는 정보 수집 담당자(신문사 스크랩 업무)를 위한 **음성 우선(voice-first) 인터페이스 프로토타입**입니다. 후천적 저시력 사용자가 시각적 정보 탐색에서 청각적 정보 탐색으로 전환할 때 겪는 어려움을 줄이는 것을 목표로 합니다.

## 통합된 4가지 설계 조각

1. **진행 단계 음성 안내** — "질문을 듣고 있습니다 → 정보를 찾는 중 → 답변 작성 중" 등 작업 진행 상태를 단계별로 음성으로 알려, 화면을 보지 못해도 시스템 상태를 파악할 수 있게 합니다.
2. **완료 신호 설계** — 검색이 끝나면 분량(몇 건), 구성(카테고리 분포), 강조(주목할 기사), 행동 안내(다음에 누를 키)를 한 번에 음성으로 전달합니다.
3. **3계층 점진적 공개** — 기사를 헤드라인 / 요약 / 전문 3계층으로 나눠, 사용자가 화살표 키로 필요한 깊이만큼만 펼쳐 듣습니다.
4. **요약 먼저 읽기** — 답변이 모두 생성되기를 기다리지 않고, 요약 부분이 만들어지는 즉시 먼저 음성으로 들려줍니다.

## 진행 단계 음성 안내

음성으로 질문하면 "질문을 듣고 있습니다 → 질문 인식 완료 → 정보를 찾는 중 → 답변 작성 중" 순으로 진행 상태를 음성으로 안내합니다. 각 단계 음성이 완전히 끝난 뒤 2초를 쉬고 다음 단계를 안내하므로, 음성이 서로 겹치지 않고 또렷하게 들립니다.

## 진입 안내 음성의 안정성

진입 화면을 누르면 시작 안내 음성이 반드시 재생됩니다. 브라우저 음성 합성의 불안정성(음성 목록 로딩 지연, cancel 직후 speak 누락, 장문 음성 멈춤)을 보정해 어떤 상태에서도 안내가 누락되지 않습니다. 구체적으로는 한국어 음성 목록이 준비된 뒤 안내를 재생하고, 음성 취소와 재생 사이에 짧은 지연을 두며, 누른 즉시 텍스트 안내를 먼저 표시해 음성 준비 중에도 반응을 보여줍니다.

## 기사 데이터에 대한 안내

이 프로토타입의 기사는 한국은행 기준금리 관련 예시 데이터입니다. 실제 뉴스를 연동하려면 언론사와의 정식 협의가 필요합니다. 언론사 RSS는 개인적 구독 용도로 한정되며, 공개 웹사이트에서 다수에게 재제공하려면 저작권자의 허락이 필요하기 때문입니다. 이 프로토타입의 목적은 음성 우선 인터페이스 자체를 검증하는 것이며, 그 목적은 예시 데이터로 충분히 달성됩니다.

## 말하기 속도 제어

- 기본 말하기 속도는 0.8배로, 또렷하게 들리도록 천천히 시작합니다.
- 시작한 뒤 음성이 빠르게 느껴지면 **화면 아무 곳이나 누르면 음성이 즉시 멈춥니다.** 속도 슬라이더를 눈으로 찾을 필요가 없습니다.
- 멈춘 뒤 다시 들으려면 `R` 키를 누르거나 기사를 다시 선택합니다.
- 상단 설정의 슬라이더로 0.6배에서 2.0배까지 속도를 세밀하게 조절할 수 있습니다.
- 버튼·슬라이더·기사 목록 등 컨트롤 조작은 멈춤 대상에서 제외되어, 정상적인 조작과 충돌하지 않습니다.

## 조작 방법

- **접속하면 화면 전체가 시작 영역**입니다. 아무 곳이나 누르거나 아무 키나 누르면 시작됩니다 — 특정 버튼을 찾을 필요가 없습니다.
- 시작하면 음성 안내가 자동으로 흐르고, 마이크 권한 안내를 거쳐 곧바로 음성 입력이 시작됩니다.
- 음성 입력이 어려우면 **예시 질문으로 진행하기** 버튼으로 대체 가능
- 기사 목록에서: `↑` `↓` 기사 이동 · `→` 더 자세히 · `←` 접기 · `S` 스크랩 · `R` 다시 듣기
- 상단 설정에서 음성 켜기/끄기, 말하기 속도, 목소리를 조절할 수 있습니다.

## voice-first 진입 설계

시각에 의존하지 않는 사용자는 "마이크 버튼이 어디 있는지 찾는" 것 자체가 장벽입니다. 이 프로토타입은 접속 즉시 화면 전체가 하나의 시작 영역이 되어, 사용자가 버튼을 조준할 필요 없이 아무 곳이나 누르면 시작됩니다.

브라우저 보안 정책상 마이크 권한 요청은 사용자의 첫 행동(클릭·키 입력) 이후에만 가능합니다. 따라서 "접속 즉시 자동으로 마이크 켜기"는 어떤 웹사이트도 할 수 없습니다. 대신 이 설계는 그 첫 행동을 "화면 아무 곳이나 누르기"로 단순화해, 버튼 찾기 장벽을 없애면서 브라우저 규칙도 지킵니다. 첫 행동 직후부터는 음성 안내와 음성 입력만으로 진행됩니다.

## 마이크 권한 접근성 안내

저시력 사용자는 브라우저가 띄우는 마이크 권한 팝업을 보기 어렵습니다. 이 프로토타입은 권한 팝업 자체는 제어할 수 없지만, 그 전후 상황을 안내합니다.

- 팝업이 뜨기 전: "잠시 후 마이크 권한 창이 나타납니다, 허용을 선택해 주세요"라고 미리 안내
- 허용된 경우: 바로 음성 인식을 시작하며 "지금 듣고 있습니다"라고 안내
- 거부된 경우: 권한을 다시 허용하는 방법과 예시 질문 대체 수단을 안내
- 음성 미지원 브라우저: Chrome·Edge 사용을 안내하고 예시 질문으로 대체

마이크 권한과 음성 인식은 https 환경에서만 작동합니다. Vercel 배포 주소는 자동으로 https이므로 정상 동작합니다.

## 기술

순수 HTML / CSS / JavaScript로 작성되었으며 빌드 과정이 없습니다. 음성 출력은 브라우저 내장 **Web Speech API**(`speechSynthesis`)를 사용합니다. 한국어 음성 지원은 브라우저·운영체제에 따라 다르며, 미지원 시 텍스트 안내로 대체됩니다.

> 참고: 이 프로토타입의 음성 인식·검색·기사 데이터는 시연을 위한 예시이며, 실제 음성 인식이나 뉴스 검색을 수행하지 않습니다.

## GitHub에 올리고 Vercel로 배포하기

### 1. GitHub 저장소 만들기

GitHub에서 새 저장소를 만든 뒤, 이 폴더에서 다음을 실행합니다.

```bash
git init
git add .
git commit -m "음성 뉴스 스크랩 접근성 프로토타입"
git branch -M main
git remote add origin https://github.com/<사용자명>/<저장소명>.git
git push -u origin main
```

### 2. Vercel로 배포하기

방법 A — 웹 대시보드:
1. [vercel.com](https://vercel.com)에 GitHub 계정으로 로그인
2. "Add New… → Project" 선택 후 위 저장소를 가져오기(Import)
3. 프레임워크 프리셋은 "Other"로 두고(빌드 설정 불필요) "Deploy" 클릭
4. 배포가 끝나면 `https://<프로젝트명>.vercel.app` 링크가 생성됩니다.

방법 B — 명령줄:
```bash
npm i -g vercel
vercel
```

정적 사이트라 별도 빌드 명령이나 환경 변수가 필요 없습니다.
